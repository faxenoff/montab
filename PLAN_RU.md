# montab — архитектура и финальные требования (as built)

> Статус: реализовано полностью, см. README для пользовательской документации.
> Фактические метрики: один exe **1,95 МБ** (NativeAOT, жёсткий тримминг),
> ~4 МБ собственной памяти, ~0% CPU в простое, ноль аллокаций в стабильном кадре.
> Платформа: `net11.0-windows` (preview) + C# 15.

Боковая панель-таскбар для Windows с постоянными живыми превью всех окон.
Док слева или справа от рабочей области, клик — переключение, наведение — лупа,
двойной клик — системное сворачивание, минимум ресурсов.

---

## 1. Ключевые технологические решения

### 1.1. Превью: DWM Thumbnail API (не захват экрана)

Ядро — `DwmRegisterThumbnail` / `DwmUpdateThumbnailProperties` / `DwmUnregisterThumbnail`.

Почему именно это, а не Windows.Graphics.Capture / DXGI Duplication:

| Свойство | DWM Thumbnail | Windows.Graphics.Capture |
|---|---|---|
| Стоимость кадра | ~0 (DWM компонует уже готовую поверхность окна, один текстурированный квад) | Копия кадра в frame pool + наша отрисовка |
| «Битрейт» при отсутствии изменений | Нулевой по построению: если окно не перерисовалось — DWM не перекомпоновывает | Кадры продолжают приходить, нужно самим отфильтровывать |
| Задержка | Нулевая (тот же композитор) | 1+ кадр |
| Доступ к пикселям | Нет (и не нужен) | Есть |
| Окна на заднем плане (перекрытые) | Работает — DWM всегда рендерит окна в offscreen-поверхности¹ | Работает¹ |
| Свёрнутые окна | Не работает | Тоже не работает |
| Обрезка источника (zoom&pan) | `rcSource` — встроено | Вручную |
| Прозрачность/затенение | `opacity` — встроено | Вручную |

¹ Кроме Chromium-браузеров с включённым occlusion tracking — они сами перестают
рендерить полностью перекрытое окно. Решение (политика реестра) — в README.

Экономия ресурсов: никакого потока/кодирования нет вообще, только композиция GPU.
Главный рычаг — **виртуализация**: превью за пределами видимости панели снимаются
с регистрации и не стоят ничего.

Два практических правила, добытых в бою:
- **`rcSource` задаётся только при активном zoom.** Прибитый rcSource фиксирует
  размер источника на момент вызова; у разворачивающегося из свёрнутого окна это
  «иконик»-полоса 160×28 → превью-блин до следующего события. Сброс zoom —
  пересозданием миниатюры (флаги DWM только добавляют поля, отменить нельзя).
- **Контент миниатюры компонуется ПОВЕРХ нашего GDI** в заданном прямоугольнике —
  рамки-подсветки и подписи рисуются вокруг превью, не поверх.

### 1.2. Платформа: C# / .NET 11 + NativeAOT + чистый Win32

- **Target**: `net11.0-windows`, `LangVersion=preview` (C# 15).
- **PublishAot + жёсткий тримминг**: `TrimMode=full`, `InvariantGlobalization`,
  `StackTraceSupport=false`, `UseSystemResourceKeys=true`, отключены
  Debugger/EventSource/Metrics-поддержка, `IlcFoldIdenticalMethodBodies`.
  GC: неконкурентный, `RetainVM=false`, `ConserveMemory=7`.
- **Без UI-фреймворка**: ни WPF (нет AOT), ни WinUI 3 (WindowsAppSDK — десятки МБ).
  Чистое Win32-окно + свой WndProc.
- **P/Invoke**: `Microsoft.Windows.CsWin32` — source generator, build-time-only
  зависимость, `allowMarshaling=false`, `useSafeHandles=false`. Список импортов
  (`NativeMethods.txt`) поддерживается точным — только реально используемое.
- **Отрисовка**: GDI с кешированным backbuffer (пересоздание только при ресайзе)
  и предвычисленными на текущий DPI размерами. Планировавшийся Direct2D не
  понадобился: тёмный фон + рамки + текст GDI закрывает полностью,
  а сами превью рисует DWM.
- **JSON-настройки**: System.Text.Json source generation (без рефлексии).

Статические импорты exe — только ОС: kernel32, advapi32 (автостарт), bcrypt,
ole32, UCRT api-sets; user32/gdi32/dwmapi/shell32 подгружаются при первом вызове.

### 1.3. Док панели: AppBar API

`SHAppBarMessage`: `ABM_NEW` → `ABM_QUERYPOS`/`ABM_SETPOS` с `ABE_LEFT`/`ABE_RIGHT`.
Система сама ужимает рабочую область — развёрнутые окна не налезают на панель.
`ABN_FULLSCREENAPP` — уход в низ z-order на время полноэкранных приложений,
`ABN_POSCHANGED` — пересчёт при изменении конфигурации мониторов.

Важно: бит `WS_EX_TOPMOST` рассинхронизируется с фактической z-позицией —
`HWND_TOPMOST` переутверждается через `SetWindowPos` при каждом размещении
(как делает системный таскбар).

### 1.4. Трекинг окон: события, не поллинг

- Стартовая инвентаризация: `EnumWindows` + классический alt-tab-фильтр
  (Raymond Chen): `WS_VISIBLE`, непустой заголовок, без `WS_EX_TOOLWINDOW`,
  корневой владелец owned-цепочки, не cloaked (`DWMWA_CLOAKED` отсекает окна
  чужих виртуальных рабочих столов).
- Дальше — `SetWinEventHook` (`WINEVENT_OUTOFCONTEXT | WINEVENT_SKIPOWNPROCESS`,
  колбэки приходят в наш UI-поток через message loop):
  - `EVENT_OBJECT_SHOW / HIDE / DESTROY`, `CLOAKED / UNCLOAKED` — состав списка;
  - `EVENT_SYSTEM_FOREGROUND` — подсветка активного + MRU-история;
  - `EVENT_SYSTEM_MINIMIZESTART / MINIMIZEEND` — полоска ⇄ живое превью
    + перестановка между секциями;
  - `EVENT_OBJECT_NAMECHANGE` — подпись; также «поздняя» регистрация окон,
    ставящих заголовок после показа;
  - `EVENT_OBJECT_LOCATIONCHANGE` — пересчёт аспекта источника (с защитой от
    «иконик»-геометрии: `IsIconic` + кламп пропорций 0.2–4.5).

## 2. Архитектура (фактическая)

Один процесс, один UI-поток с message loop, одно главное окно-панель.

```
montab/
├─ src/
│  ├─ Program.cs              # входная точка, single-instance mutex, message loop
│  ├─ App/
│  │  ├─ PanelWindow.cs       # HWND, WndProc, вся state machine взаимодействий,
│  │  │                       #   док/ресайз/перенос панели, контекстное меню, DPI-кеш
│  │  ├─ AppBar.cs            # обёртка SHAppBarMessage
│  │  └─ Strings.cs           # строки меню RU/EN (GetUserDefaultUILanguage)
│  ├─ Core/
│  │  ├─ WindowTracker.cs     # EnumWindows + WinEventHook → двухсекционный список
│  │  ├─ WindowItem.cs        # hwnd, title, icon, аспект, IsMinimized, zoom/center
│  │  └─ SwitchController.cs  # MRU-история foreground + активация с обходами
│  ├─ Thumbs/
│  │  └─ ThumbnailManager.cs  # DWM-миниатюры: виртуализация, rcSource, opacity
│  ├─ UI/
│  │  ├─ LayoutEngine.cs      # раскладка (переиспользуемый буфер, DPI-кеш)
│  │  └─ Renderer.cs          # GDI: backbuffer, рамки, подписи, крестик, гриппер
│  └─ Config/
│     └─ Settings.cs          # STJ source-gen; %APPDATA%\montab\settings.json
└─ montab.csproj              # net11.0-windows, PublishAot, тримминг, CsWin32
```

### Модель данных: двухсекционная лента

`WindowItem`: hwnd, title, icon, аспект источника, `IsMinimized`, zoom/center.
Единый упорядоченный список с инвариантом: **живые превью сверху, свёрнутые
полоски снизу**. Свернувшееся окно встаёт на границу секций (первым среди
полосок), развернувшееся — на ту же границу (последним среди живых) — это одна
и та же операция вставки. Новые живые окна — в самый верх, новые свёрнутые —
первыми среди полосок. Планировавшееся отдельное состояние «погашен» (Collapsed)
упразднено: сворачивание из панели — это настоящий системный minimize.

### Раскладка

- Ширина панели: clamp **3–20%** ширины монитора, ресайз за внутренний край.
- Сверху панели — «ручка» (14 лог. px, гриппер-точки) для переноса панели.
- Живой тайл: заголовок (18 лог. px: иконка + название + крестик ✕) над превью;
  высота превью = ширина/аспект с клампом; DWM-прямоугольник вписывается
  по центру (сам DWM прижимает к левому верху).
- Полоска: 22 лог. px, иконка + название + крестик.
- Gap 6 лог. px; все размеры предвычисляются на DPI монитора панели.
- Скролл колесом, кламп по суммарной высоте.

### Переключение окон (финальная семантика)

- **Клик по живому тайлу** (любое место): активация с задержкой 150 мс —
  окно ожидания второго клика.
- **Двойной клик по живому тайлу**: системный minimize (`SW_SHOWMINNOACTIVE`).
  Второй клик детектируется **по второму нажатию** (DOWN), не по отпусканию:
  реальный дабл держит кнопку ~80–100 мс, и окно UP→UP не влезает в разумную
  задержку, а UP→DOWN — влезает. Отпускание второго клика глотается.
  Системный `WM_LBUTTONDBLCLK` не используется (лента перестраивается после
  первого клика, 4-пиксельная системная зона промахивается).
- **Клик по полоске**: мгновенный restore + активация; двойной клик эквивалентен
  одинарному (второй клик — no-op).
- После сворачивания активного окна фокус уходит в **последнее по MRU-истории
  открытое окно**, свёрнутые и закрытые пропускаются (история — 32 записи,
  фильтр задаёт панель). Семантика «повторный клик по активному → предыдущее
  окно» пробовалась и убрана как неудобная.
- Панель `WS_EX_NOACTIVATE` + topmost: клики не воруют фокус. Обход
  foreground-lock: `keybd_event(VK_MENU)` при отказе `SetForegroundWindow`.

### Zoom&Pan (финальная семантика)

- **Hover-лупа**: наведение на превью ~0,7 с → временный zoom **×5** (ровно,
  не поверх постоянного), движение мыши панорамирует (курсор SIZEALL),
  уход с превью / скролл / начало drag — восстановление прежнего состояния.
  Клик отменяет «намерение» лупы — мерцания при переключениях нет.
- **Постоянный zoom**: Ctrl+колесо ×1–5 (мультипликативные шаги), Ctrl+движение —
  pan, Ctrl+клик — сброс. Реализация: `rcSource` = фрагмент 1/zoom вокруг
  нормализованного центра.
- Планировавшийся press-and-hold zoom удалён — конфликтовал с drag-reorder.

### Drag-reorder и перенос панели

- Сдвиг > 8 лог. px от нажатия → перетаскивание тайла: live-перестановка списка,
  курсор ↕, таскаемый элемент подсвечен (светлее фон + серый outline).
  Только в пределах своей секции. Порядок между запусками не сохраняется.
- Перетаскивание за «ручку» или пустую зону → перенос панели: бросок на любую
  половину любого монитора докает панель на соответствующий край (курсор ✥).

### Прочее

- Крестик ✕ в каждом заголовке: hover — красный, клик — `WM_CLOSE` окну;
  повторный клик в той же точке игнорируется (лента сдвинулась — под курсором
  чужой крестик).
- Активное окно: акцентная рамка вокруг всего блока + затенение превью
  (`opacity` ≈ 110).
- Контекстное меню: край дока, автозапуск (HKCU\...\Run), выход; локализовано
  (русский язык системы → русский, иначе английский).
- Настройки (край, ширина, монитор) сохраняются при каждом изменении
  и на `WM_ENDSESSION`; восстанавливаются при старте (пропавший монитор →
  primary).
- Per-Monitor V2 DPI: физические пиксели везде, DPI-зависимые размеры
  предвычисляются при `WM_DPICHANGED`.

## 3. Отклонения от исходного плана

| Планировалось | Стало | Почему |
|---|---|---|
| Direct2D/DirectWrite | GDI + кешированный backbuffer | Превью рисует DWM; для фона/рамок/текста D2D избыточен |
| Повторный клик по активному → предыдущее окно | Убрано | Неудобный UX (подтверждено практикой) |
| Zoom ×3 по удержанию кнопки | Hover-лупа ×5 + Ctrl-режимы | Удержание конфликтовало с drag-reorder |
| Состояние «погашен» (Collapsed) отдельно от minimize | Единое: полоска = системно свёрнутое окно | Две механики путались; системный minimize честнее |
| Заголовок под превью | Заголовок над превью | Просьба по результатам использования |
| Отложенная активация ~500 мс (системный dblclick) | 150 мс + детект второго клика по DOWN | Системная зона 4 px промахивалась из-за перестройки ленты |
| Сохранение порядка между запусками | Не сохраняется | Порядок «живёт» с окнами; ценность не подтвердилась |
| Трей-иконка | Нет | Контекстное меню панели покрывает |
| WGC-детектор «активности» полоски | Нет | Дорого; события (restore/foreground) достаточны |

## 4. Риски: что подтвердилось

| Риск из плана | Итог |
|---|---|
| `SetForegroundWindow` без активации панели | Подтвердился частично; хватает Alt-трюка |
| WS_EX_TOPMOST теряется | Подтвердился; лечится SetWindowPos(HWND_TOPMOST) при каждом размещении |
| DWM поверх нашей графики | Подтвердился; рамки рисуются вокруг превью |
| Chromium перестаёт рендерить перекрытое окно | Новый, не предвиделся; документирован в README (политика occlusion) |
| «Иконик»-геометрия при разворачивании | Новый; лечится незакреплённым rcSource + фильтром аспекта |
| Виртуальные рабочие столы | Как планировалось: показывается текущий стол (cloaked отсекаются) |

## 5. Критерии готовности — факт

- Один exe (NativeAOT) **1,95 МБ**, внешних зависимостей ноль. ✓
- ~4 МБ private memory; CPU ≈ 0% в простое; GPU — только композиция DWM. ✓
- Превью обновляются с частотой источника без видимой задержки. ✓
- Аспект, gap, клик-переключение, двойной клик-сворачивание, drag-reorder,
  двухсекционная лента, скролл + виртуализация, hover-лупа ×5 и Ctrl-zoom ×1–5,
  подсветка активного, док 3–20%, перенос между мониторами, PMv2 DPI,
  автозапуск, сохранение настроек. ✓
