# montab — план реализации

Боковая панель-таскбар для Windows с постоянными живыми превью всех окон.
Док слева или справа от рабочей области, клик — переключение, повторный клик — возврат
в предыдущее окно, zoom&pan по удержанию, минимум ресурсов.

---

## 1. Ключевые технологические решения

### 1.1. Превью: DWM Thumbnail API (не захват экрана)

Ядро — `DwmRegisterThumbnail` / `DwmUpdateThumbnailProperties` / `DwmUnregisterThumbnail`.

Почему именно это, а не Windows.Graphics.Capture / DXGI Duplication:

| Свойство | DWM Thumbnail | Windows.Graphics.Capture |
|---|---|---|
| Стоимость кадра | ~0 (DWM компонует уже готовую поверхность окна, один текстурированный квад) | Копия кадра в frame pool + наша отрисовка |
| «Битрейт» при отсутствии изменений | Нулевой по построению: если окно не перерисовалось — DWM не перекомпоновывает | Кадры продолжают приходить, нужно самим отфильтровывать |
| Задержка | Нулевая (тот же композитор) | 1+ кадр |
| Доступ к пикселям | Нет (и не нужен) | Есть |
| Окна на заднем плане (перекрытые) | Работает — DWM всегда рендерит окна в offscreen-поверхности | Работает |
| Свёрнутые окна | Не работает | Тоже не работает |
| Обрезка источника (zoom&pan) | `rcSource` — встроено | Вручную |
| Прозрачность/затенение | `opacity` — встроено | Вручную |

Требование «экономить битрейт при малых изменениях» DWM закрывает автоматически:
никакого потока/кодирования нет вообще, есть только композиция GPU. Главный рычаг
экономии — **виртуализация**: превью за пределами видимости панели снимаются с
регистрации (`DwmUnregisterThumbnail`) и не стоят ничего.

Свёрнутые окна DWM не показывает — по ТЗ они и так «полоски», конфликта нет.

Ограничение DWM: контент миниатюры рисуется ПОВЕРХ содержимого нашего окна в заданном
прямоугольнике. Поэтому рамки-подсветки, подписи и gap рисуем ВОКРУГ прямоугольника
превью, не поверх него. Дизайн это учитывает.

### 1.2. Платформа: C# / .NET + NativeAOT + чистый Win32

- **Target**: `net10.0-windows` (актуальный LTS; на .NET 11 перейти одной строкой,
  когда выйдет из preview в ноябре 2026).
- **PublishAot + полный тримминг**, `InvariantGlobalization`, без рефлексии.
  Ожидаемо: один exe ~3–6 МБ, RAM ~20–40 МБ, CPU в простое ~0%.
- **Без UI-фреймворка**: ни WPF (нет AOT), ни WinUI 3 (WindowsAppSDK — десятки МБ).
  Чистое Win32-окно + свой WndProc. UI панели прост: фон, рамки, текст, иконки.
- **P/Invoke**: `Microsoft.Windows.CsWin32` — source generator, build-time-only
  зависимость, AOT-совместимый (COM без маршалинга).
- **Отрисовка**: Direct2D + DirectWrite (HwndRenderTarget) — чёткий DPI-aware текст,
  скруглённые рамки, дешёвая перерисовка при скролле. COM-интероп через CsWin32
  (`allowMarshaling: false`) либо TerraFX.Interop.Windows (blittable, AOT).
  Запасной упрощённый вариант — GDI: для тёмного фона + текста его хватает.
- **JSON-настройки**: System.Text.Json с source generation (без рефлексии).

Zig дал бы exe ~300 КБ, но выигрыш непринципиален (вся тяжесть — в DWM, не в нас),
а скорость разработки и примеры (OnTopReplica) — на стороне .NET.

### 1.3. Док панели: AppBar API

`SHAppBarMessage`: `ABM_NEW` → `ABM_QUERYPOS`/`ABM_SETPOS` с `ABE_LEFT`/`ABE_RIGHT`.
Система сама ужимает рабочую область — развёрнутые окна не налезают на панель.
Это тот же механизм, которым пользуется системный таскбар; корректно уживается
с другими appbar'ами. Обрабатываем `ABN_FULLSCREENAPP` (уход за полноэкранные
приложения), `ABN_POSCHANGED` (пересчёт при изменении конфигурации мониторов).

### 1.4. Трекинг окон: события, не поллинг

- Стартовая инвентаризация: `EnumWindows` + классический alt-tab-фильтр:
  `WS_VISIBLE`, без `WS_EX_TOOLWINDOW`, без владельца (или с `WS_EX_APPWINDOW`),
  не cloaked (`DwmGetWindowAttribute(DWMWA_CLOAKED)` — отсекает окна чужих
  виртуальных рабочих столов).
- Дальше — `SetWinEventHook` (`WINEVENT_OUTOFCONTEXT`, работает без DLL-инъекций):
  - `EVENT_OBJECT_SHOW / HIDE / DESTROY` — появление/уход окон;
  - `EVENT_SYSTEM_FOREGROUND` — подсветка активного + история для «возврата»;
  - `EVENT_SYSTEM_MINIMIZESTART / MINIMIZEEND` — полоска ⇄ живое превью;
  - `EVENT_OBJECT_NAMECHANGE` — обновление подписи; заодно дешёвая эвристика
    «в свёрнутом окне началась активность»;
  - `EVENT_OBJECT_LOCATIONCHANGE` (фильтр по нашему набору hwnd, с троттлингом) —
    изменение размера источника → пересчёт аспекта и `rcSource`.

## 2. Архитектура

Один процесс, один UI-поток с message loop (нужен и для WinEventHook), одно
главное окно-панель. Никаких сервисов и вспомогательных процессов.

```
montab/
├─ src/
│  ├─ Program.cs              # входная точка, single-instance mutex, message loop
│  ├─ App/
│  │  ├─ PanelWindow.cs       # главный HWND, WndProc, роутинг сообщений
│  │  ├─ AppBar.cs            # обёртка SHAppBarMessage, резервирование края
│  │  └─ Dpi.cs               # PMv2, WM_DPICHANGED, physical↔logical px
│  ├─ Core/
│  │  ├─ WindowTracker.cs     # EnumWindows + WinEventHook → события модели
│  │  ├─ WindowItem.cs        # hwnd, title, icon, аспект, состояние (live/strip)
│  │  └─ SwitchController.cs  # переключение foreground + история «туда-обратно»
│  ├─ Thumbs/
│  │  ├─ DwmThumbnail.cs      # register/update/unregister, rcSource, opacity
│  │  └─ ThumbnailManager.cs  # виртуализация: видимые ↔ спящие превью
│  ├─ UI/
│  │  ├─ LayoutEngine.cs      # раскладка тайлов/полосок, скролл, hit-test
│  │  ├─ Renderer.cs          # Direct2D: фон, рамки, подписи, иконки
│  │  └─ Interaction.cs       # state machine: click / drag / zoom&pan / resize
│  └─ Config/
│     └─ Settings.cs          # STJ source-gen; %APPDATA%\montab\settings.json
└─ montab.csproj              # PublishAot, TrimMode=full, CsWin32
```

### Модель данных

`WindowItem`: hwnd, pid, title, hIcon, аспект источника, состояние
(`Live | Strip | Collapsed`), позиция в ленте. Список упорядочен вручную,
новые окна вставляются сверху. `Collapsed` — «погашенные» пользователем
(отличие от `Strip`-свёрнутых: окно живо, но превью отключено нами).

### Раскладка

- Ширина панели: настраиваемая, clamp **3%–20%** ширины монитора; ресайз —
  захватом внутреннего края (курсор ⇔, `WM_SETCURSOR` + свой хит-тест).
- Живой тайл: ширина = ширина панели − 2×padding; высота = ширина / аспект окна
  (аспект сохраняется всегда); ниже — подпись (иконка + заголовок, 1 строка).
- Полоска (`Strip`/`Collapsed`): высота ~28 px (лог.), иконка + название.
- Gap между элементами ~6 px (лог.), всё масштабируется по DPI монитора панели.
- Скролл колесом: смещение viewport'а; полоса прокрутки не нужна (тонкий
  индикатор — опционально).

### Виртуализация превью

`ThumbnailManager` после каждого изменения layout/скролла сверяет видимый
диапазон: тайлам в viewport (+1 экран запаса — опционально) — регистрация
DWM-превью; ушедшим — `DwmUnregisterThumbnail`. Регистрация дешёвая,
перекатывание при скролле незаметно.

### Переключение окон

- Клик по тайлу/полоске:
  - если hwnd ≠ foreground → `ShowWindow(SW_RESTORE)` при необходимости +
    `SetForegroundWindow`;
  - если hwnd == foreground → переключаемся на **предыдущее** foreground-окно
    (история в `SwitchController`, обновляется по `EVENT_SYSTEM_FOREGROUND`).
- Панель — `WS_EX_NOACTIVATE` + `WS_EX_TOPMOST`: клики не воруют фокус.
  Известная ловушка: без активации может не хватить прав на
  `SetForegroundWindow`. Отработанные обходы (по нарастающей):
  клик по нашему окну даёт последний ввод → обычно достаточно;
  иначе — `AllowSetForegroundWindow`/трюк с `keybd_event(VK_MENU)`.
  Закладываем как отдельную задачу с тестами (включая elevated-окна).

### Активное окно

- Подсветка: акцентная рамка вокруг тайла (рисуем вокруг прямоугольника DWM).
- Затенение: `opacity ≈ 120` у DWM-превью. Управлять FPS у DWM нельзя, но и
  незачем — стоимость нулевая. Опция «заморозить» (снять регистрацию + статичный
  снимок через `PrintWindow(PW_RENDERFULLCONTENT)`) — в бэклог, если захочется.

### Zoom&Pan (удержание кнопки)

State machine в `Interaction.cs`:

- `MouseDown` → ждём: сдвиг > drag-порога раньше 300 мс → **drag-reorder**;
  удержание 300 мс без сдвига → **zoom-режим**; отпускание раньше → **клик**.
- Zoom: `rcSource` = прямоугольник размером (W/3 × H/3) источника с центром в
  точке, соответствующей курсору; dest — весь тайл ⇒ честное увеличение ×3.
  Движение мыши панорамирует (позиция курсора в тайле → позиция окна источника,
  с клампом по краям). Отпустили — вернули `rcSource` на всё окно, клик не
  срабатывает. `fSourceClientAreaOnly = true`, чтобы не увеличивать рамки окна.

### Drag-reorder

Порог сдвига → режим перетаскивания: тайл следует за курсором (рисуем
плейсхолдер), список переупорядочивается по отпусканию. Порядок сохраняется
в настройках (ключ — exe + нормализованный заголовок, best effort).

### Полоски и «оживление»

- Свёрнутое окно → `Strip` автоматически (`MINIMIZESTART`); разворачивание
  (`MINIMIZEEND`) → снова живой тайл.
- Ручное «погасить» → `Collapsed`: средняя кнопка мыши или пункт контекстного
  меню; превью снимается с регистрации (ресурсы = 0). Возврат — клик
  (активация окна автоматически оживляет тайл).
- «Началась активность» у полоски: дешёвые сигналы — смена заголовка
  (`NAMECHANGE`), выход из свёрнутого состояния, foreground. Пиксельный
  frame-diff потребовал бы Windows.Graphics.Capture — сознательно не делаем в v1.

### Мультимонитор и DPI

- Манифест: **Per-Monitor DPI Awareness V2**. Все прямоугольники — физические
  пиксели; логические размеры (gap, полоски, шрифты) умножаются на DPI монитора,
  где стоит панель. `WM_DPICHANGED` → полный пересчёт layout.
- Перенос панели: drag за «шапку»/пустую зону → ghost-контур; бросок у
  левого/правого края любого монитора → передок appbar'а туда
  (`ABM_SETPOS` с новым rect), сохранение в настройках.
- Панель показывает окна **всех** мониторов независимо от того, где стоит.
- Изменение конфигурации мониторов (`WM_DISPLAYCHANGE`, `ABN_POSCHANGED`) →
  ревалидация: если монитор панели исчез — переезд на primary.

## 3. Этапы реализации

Каждый этап — работающий инкремент.

**M1. Каркас панели** (фундамент)
Win32-окно, message loop, PMv2-манифест, AppBar-док слева/справа, резервирование
рабочей области (проверка: maximized-окна не налезают), ресайз края с clamp
3–20%, сохранение настроек. Тёмный фон, Direct2D поднят.

**M2. Трекинг окон**
`WindowTracker`: enum + alt-tab-фильтр + WinEventHook. Модель списка, новые —
сверху. Пока рисуем просто полоски с иконкой (`SHGetFileInfo` /
`IShellItemImageFactory`) и заголовком для всех окон.

**M3. Живые превью**
`DwmThumbnail` + `ThumbnailManager`, раскладка тайлов с аспектом и gap'ами,
полоски для свёрнутых. Реакция на resize источника (пересчёт аспекта).

**M4. Переключение и подсветка**
`SwitchController`: клик → foreground, повторный клик → предыдущее окно.
Обходы SetForegroundWindow-lock. Акцентная рамка + затенение активного.

**M5. Скролл и виртуализация**
Колесо мыши, viewport, unregister невидимых превью. Проверка на 40–60 окнах.

**M6. Drag-reorder и ручное сворачивание**
State machine взаимодействий (клик/драг разведены порогами), средняя кнопка /
контекстное меню → `Collapsed`, «оживление» по событиям.

**M7. Zoom&Pan**
Ветка удержания в state machine, `rcSource`-увеличение ×3, панорамирование.

**M8. Мультимонитор**
Перенос панели drag'ом, WM_DPICHANGED/WM_DISPLAYCHANGE, ABN_FULLSCREENAPP,
исчезновение монитора.

**M9. Полировка и упаковка**
Трей-иконка (выход/настройки/автостарт), автостарт (HKCU\...\Run), publish
NativeAOT (замер размера/памяти), профилирование GPU (PresentMon/Task Manager),
edge-cases: elevated-окна, UWP/cloaked, виртуальные рабочие столы.

## 4. Риски и открытые вопросы

| Риск | Митигация |
|---|---|
| `SetForegroundWindow` без активации панели может отклоняться | Каскад обходов (M4), тесты на elevated и UWP-окнах |
| WinEventHook и elevated-окна (UIPI) | Out-of-context события обычно доходят; проверить в M2; при проблемах — опция запуска панели elevated |
| UWP/cloaked окна: превью пустое у приостановленных приложений | Детект по `DWMWA_CLOAKED` → показывать полоску вместо чёрного тайла |
| Виртуальные рабочие столы: окна других столов cloaked | v1: показываем только текущий стол; интеграция с IVirtualDesktopManager — бэклог |
| Direct2D через CsWin32 без маршалинга — заметный объём интеропа | Fallback: GDI-рендер (фон/текст/рамки), D2D добавить позже |
| DWM-превью нельзя перекрыть нашей графикой | Дизайн: рамки/подписи вокруг превью, не поверх |
| Много окон (>60) | Виртуализация обязательна с M5; полоски бесплатны |
| «Оживление» полоски без пиксельного анализа | Эвристики по событиям; WGC-детектор активности — осознанно вне scope v1 |

## 5. Критерии готовности v1

- Один exe (NativeAOT), ≤ 8 МБ, без внешних зависимостей.
- RAM ≤ 50 МБ при 30 окнах; CPU ≈ 0% в простое; GPU-нагрузка неотличима от фона.
- Превью обновляются с частотой источника без видимой задержки.
- Все пункты ТЗ: аспект, gap, клик/возврат, drag-reorder, полоски, новые сверху,
  скролл + виртуализация, zoom&pan ×3, подсветка активного, док 3–20%,
  мультимонитор, PMv2 DPI.
